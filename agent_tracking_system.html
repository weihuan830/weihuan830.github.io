Agent Tracking System

AI Intern Agent - 智能研究助手系统

核心理念：我们以往说到过希望有一个 AI的 Intern Agent，AI来辅助我们做Research Check，这个Agent Tracking System，就是 Intern Agent

1. 为什么叫做 Agent Tracking System？

Answer： 重点在于Tracking，Intern（包括Analyst）很多时候做的事情，是长期追踪某一个公司的业务相关的问题和问题背后的答案。PM在得到某个事情的持续性输出后，会学习到新的事情，建立conviction，做出判断。

因此，AI能辅助长期Track一件事情至关重要。

2. Intern Agent 的三个关键要素

做一个 Intern Agent 辅助做 Research Check，有三个关键点：

📊 Data

高质量、实时、相关的数据源

🤖 AI Model

强大的AI模型进行分析和理解

❓ Prompt

精准的research questions

3. 最初的尝试与挑战

我们最初尝试过类似Manus和Deep Research的方式，在互联网直接搜索相关的文档，但遇到了以下问题：

  • Data的实效性、相关性无法保障
  • 自动生成的research questions和分析师想要的方向不同
  • AI输出的结果无法做任何保障 - Idea Too Ambitious to use
  • 无法让分析师对每个问题都写有效的prompt，拆解问题本身也是一个问题

核心挑战：知道自己的问题是什么，并且知道AI + 数据可以完成什么内容，是非常困难的

实现内容：

数据来源：

  • 行业公司的垂直分类新闻网站
  • 公司主页等
  • 我们以往的爬虫有数据积累

与Manus的区别：我们相当于是指定了数据集，数据是可控的、可解释的数据集

Prompt优势：这些报告的prompt正好我也可以写，我知道主要的research question是什么，并且因为是长期追踪的内容，我也方便和分析师讨论迭代

5. 数据成为限制的原因

在此MVP基础上，我发现数据本身逐渐成为限制，理由如下：

  • 一些分析师能看到的sellside portal report，AI获取不到，会影响分析师的conviction
  • 一些行业的public information不够
  • 中小型公司的public information不够

6. AI是怎么生产report的？

核心逻辑：非常简单的逻辑，使用数据回答问题，数据量很大的时候分块回答，再合并。

从分析师的角度，容易理解；报告的角度，容易解释。

Agent Tracking System 的核心原则

系统运作的三大原则：

  • 我们长期对一个research相关的问题进行追踪
  • 需要Analyst提出合适的问题，以及我找到能回答问题的数据
  • 如果分析师所见的数据，就是AI所见的数据，那么AI去辅助解决问题就会有一定程度的conviction
